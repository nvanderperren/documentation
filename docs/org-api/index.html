meemoo Organization API V2

Documentatie voor meemoo's Organization API V2. Deze API geeft je toegang tot informatie over organisaties en personen binnen de meemoo context, zoals contentpartners, educatieve partners, vestigingen, contact informatie, account managers, or personeelsleden van meemoo. Het is een GraphQL API die voldoet aan de specificatie. Een GraphQL query kan zowel met GET of POST verstuurd worden.

https://services-qas.viaa.be/organisation_v2
Version: 2.0.0
GET request

Een GraphQL GET request bevat de query in de query query string.
https://services-qas.viaa.be/organisation_v2?query=<graphql_query>

POST request

Een GraphQL POST request verwacht een JSON-gecodeerde payload. De request moet daarom de header Content-Type: application/json bevatten. De payload is als volgt gestructureerd:

{
  "query": "...",
  "operationName": "...",
  "variables": { "myVariable": "someValue", ... }
}
Testing

De QAS API bevat ook een interactieve GraphiQL omgeving waar queries kunnen getest worden. Je kan deze vinden door https://services-qas.viaa.be/organisation_v2 in je browser te openen.

Authentication

Deze API gebruikt authenticatie via JSON Web Tokens (JWT). Het JWT token kan je als volgt aanvragen met je meemoo gebruikersnaam en paswoord.

  1. Vraag een token aan via 
    curl -v -u '<username/email/...>:<password>' "https://oas-qas.meemoo.be/token?grant_type=client_credentials&expires_in=31536000"
  2. Gebruik het token in de authorization header (bearer) of uri parameter (jwauth):
    curl -v https://services-qas.viaa.be/organisation_v2/?jwtauth="eyJ0eXAiOiJKV1......tbjz4-r4"

ID

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.
